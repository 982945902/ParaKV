# 背景
在线推理服务（如推荐、广告系统）中，模型参数量已从十亿级向万亿级演进，存储规模向TB级别扩展，计算节点（Worker）与参数服务器（PS）之间的数据交换也成为传输瓶颈。  
LLM KVCache存储规模巨大，同样需要TB甚至PB级别的存储规模，并提供高性能的数据传输方案。  
RDMA（Remote Direct Memory Access）技术通过内核旁路、零拷贝、CPU卸载等特性，可实现微  秒级延迟和数百Gbps吞吐，成为构建高性能分布式存储的理想选择。设计方案中需要充分利用RDMA的技术，推理场景提供高性能的传输方案。
NVMe SSD使用PCIe通道和NVMe协议的SSD（常为M.2形式），速度远超SATA（通常超过3000MB/s，最快可达7000MB/s以上），常见容量规模3.84TB、7.68TB、15.36TB等，非常适合做DDR内存的扩展。同时，基于NVMe-OF协议，可提供基于RDMA的CPU卸载的访问，为构建高性能存储提供基础。  
# 设计目标
提供基于RDMA传输协议，支持NVMe扩展的高性能、高可用、可扩展的KV存储系统ParaKV（Parallel KV for LLM & Recommendation at Storage Speed），能够支持传统推荐的稀疏参数以及LLM KVCache的存储。
* **高性能**：微秒级延迟，单机千万级QPS，支持高吞吐批量读取。
* **高可用**：99.99%服务可用性，容忍单点故障、网络分区，支持无缝切换和版本回滚。
* **可扩展**：支持10TB级模型参数，通过分层存储（DRAM+SSD）突破内存限制；支持PB级别的KVCache存储系统。
* **易集成**：兼容主流RoCE网卡，无缝融入现有brpc微服务框架。
# RDMA技术概述与选型
## RDMA核心优势
* **内核旁路**：应用直接与网卡交互，避免内核协议栈开销。
* **零拷贝**：数据直接在应用内存间传输，无需中间缓冲。
* **CPU卸载**：网卡硬件处理传输协议，CPU几乎零参与。
* **高带宽低延迟**：现代RDMA网卡可达200Gbps，延迟低至1μs。
## RDMA协议与硬件
本方案采用RoCEv2（RDMA over Converged Ethernet）协议，因其基于标准以太网，部署成本低，且兼容现有IP网络。硬件需支持RoCE的网卡（如Mellanox ConnectX系列、Broadcom NetXtreme-E、华为HNS等）及支持PFC/ECN的交换机。
## 核心概念
* **Memory Region (MR)**：注册的内存区域，网卡可直接访问，通过l_key/r_key保护。
* **Queue Pair (QP)**：通信端点，包含发送队列和接收队列。
* **Completion Queue (CQ)**：通知操作完成的队列。
* **单边操作**：RDMA Read/Write，目标端CPU不参与。
* **双边操作**：Send/Recv，类似传统消息传递，目标端CPU需参与。
## 编程挑战与封装
原生Verbs API复杂，需显式管理QP、MR、CQ等资源。本方案设计Opt-PS-RDMA通信库，封装底层细节，提供类似RPC的pull/push接口，并集成连接池、内存池、故障切换等能力。
# NVMe SSD技术及相关优化
在ParaKV的设计中，选择 NVMe SSD 作为核心存储介质是构建高性能系统的基石。我们将从协议特性、软件栈驱动、应用布局以及硬件卸载四个维度，深入阐述如何基于 NVMe SSD 进行全栈优化。
## 核心存储介质：为什么是 NVMe？
与为机械硬盘设计的传统协议（如 SATA/AHCI）不同，NVMe（Non-Volatile Memory Express，非易失性存储器标准）是专门为 SSD 设计的高性能协议。其主要优势包括：
* **极高的并行度**：传统 SATA 协议仅支持单个命令队列，且队列深度仅为 32。而 NVMe 支持高达65,535 个 I/O 队列，每个队列深度同样可达 65,535。这意味着 NVMe SSD 能完美匹配现代多核 CPU 的架构，实现真正的并行 I/O 处理。
* **低延迟与直连**：NVMe 协议直接通过PCIe总线与 CPU 通信，旁路了传统的存储控制器开销。结合 SPDK 驱动，可实现微秒级的延迟。
* **高效的指令集**：NVMe 协议精简且高效，减少了 CPU 在 I/O 路径上的指令开销。
## 软件栈优化：SPDK与用户态轮询
虽然 NVMe 硬件性能强大，但传统的 Linux 内核 I/O 栈（含系统调用、中断处理、数据拷贝）会成为性能瓶颈。ParaKV将采用SPDK来彻底解决这一问题：
* **用户态驱动**：SPDK 实现了 NVMe 驱动的用户态版本，允许ParaKV绕过内核，直接通过 MMIO 与 NVMe 设备通信，消除了系统调用上下文切换的开销。
* **轮询模式**：在高负载下，传统的硬件中断开销巨大。SPDK 采用轮询模式，让 CPU 核心主动“忙等”在设备队列上。虽然会增加功耗，但能显著降低延迟并提升吞吐量，这对于追求极致性能的 KV 存储是必要的。
* **零拷贝**：SPDK 框架支持从用户态应用直接提交 DMA 指令，数据可以在 NVMe 设备和应用程序内存之间直接传输，无需经过内核缓冲区。
## 应用层布局优化：Segment与对齐
写ParaKV的分段架构在设计上充分考虑了 NVMe SSD 的物理特性，以规避“写放大”问题并提升效率：
* **对齐 I/O**：SSD 的读写以页（Page，通常 4KB/16KB）为单位，擦除以块（Block，通常 2MB/4MB）为单位。ParaKV的 Segment 大小与 Slot 大小均严格对齐到 SSD 的页边界，确保每次读写都是对齐的原子操作，避免读-修改-写带来的额外开销。
* **Append-Only 与 GC**：ParaKV采用日志结构的 Append-Only 写入方式。这符合 NVMe 设备“顺序写”性能远优于“随机写”的特性，能够最大限度利用 SSD 的内部并行度。通过后续的 Compaction（压实）操作回收空间，将随机写转化为顺序写，显著降低写放大系数。
## 硬件特性协同：磨损均衡与玄学优化
从硬件与固件层面，NVMe SSD 的内部管理机制与ParaKV的设计相辅相成：
* **磨损均衡**：NAND 闪存的擦写次数有限。现代 NVMe 主控通过动态与静态磨损均衡算法均匀分布写入负载。ParaKV的全局 Compaction 机制使得数据在盘内持续移动，这种热度感知的数据迁移策略能够更好地配合主控的磨损均衡算法，延长 SSD 寿命。
* **TRIM 指令**：当ParaKV删除 Key 或 Compaction 释放 Segment 时，系统将主动下发TRIM指令。这能提前告知 SSD 主控哪些逻辑地址的数据已失效，帮助主控进行高效的垃圾回收，减少未来的写入延迟。
* **NVMe-oF 硬件卸载**：在分布式部署中，ParaKV将利用支持NVMe over Fabrics的智能网卡（如 NVIDIA BlueField DPU）。最新的技术甚至支持将 NVMe 数据传输路径完全卸载到硬件，绕过 Arm 核心，实现 GPU/CPU 内存与远端 NVMe SSD 之间的直接数据通信，真正实现数据通路零 CPU 参与。   

通过以上从协议、驱动、应用逻辑到硬件固件的全栈协同设计，ParaKV能够充分释放现代 NVMe SSD 的潜力，在提供内存级延迟的同时，承载海量 KV 数据。
